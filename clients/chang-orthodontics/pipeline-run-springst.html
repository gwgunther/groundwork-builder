Part 2 — Creative upgrades added this session

Creative Director (Phase 2e) New

scripts/pipeline/lib/ai-director.js · Claude Opus 4.7

Between silver and injection, a new phase emits a design DNA object with enum-constrained choices:

  • archetype (8 options): editorial-asymmetric, centered-classic, magazine-split, minimal-brutalist, warm-editorial, card-heavy, poster-hero, bold-serif-driven
  • heroVariant (5 real component branches now wired): centered, asymmetric-left, asymmetric-right, split-image, full-bleed, poster
  • servicesVariant: cards-3up, editorial-list, accordion
  • sectionOrder: dynamic — stat-bar, gallery, reviews only included if data exists
  • density / motion / radius / card treatment
  • borrowedFrom + divergenceRationale: forced reference to library memory
Key rule enforced in code: sections whose data is null are stripped from sectionOrder, not rendered with dashes. No more "—" fallbacks shipping to production.

Image Role Classifier (Phase 3e) New

scripts/pipeline/lib/ai-image-roles.js · Claude Haiku 4.5 Vision

After images download, Vision classifies every one into a semantic role (hero-office, doctor-portrait, team-group, patient-smile, interior, decorative, logo) with a confidence score.

Role assignment rule: highest-confidence wide interior → hero; doctor portraits → doctor role; rest cascades to team / interior / gallery. Output: public/images/image-roles.json consumed by index.astro at build time.

Spring St. run result: 17/17 classified, hero = heroes/office-1.jpg — the empty hero from the previous run is now filled.

distill-design + design library New

scripts/pipeline/lib/distill-design.js + _memory/library/

Takes any reference (URL or local Astro repo), compresses it to a ~1–2 KB JSON fingerprint, stores it in _memory/library/, and updates index.json. Three tags:

  • own — our recent builds (used as divergence memory / anti-inspo)
  • inspo — curated references to borrow traits from
  • anti — explicit do-not-echo list

Every shipped build auto-distills itself into the library, tagged own. The next run's Director sees it and must diverge.

Part 3 — The agentic roadmap (what's next)

The call: stop patching the linear pipeline. Build a Designer Agent with tool-use, where the skills library and pipeline utilities are its tools. Phase 2e as it exists today (one-shot DNA) is a stepping stone, not the destination.

Architecture — Designer Agent Planned

┌──────────────────────────────────────────────┐ │ DESIGNER AGENT (Claude Opus, tool-use) │ │ │ │ Inputs: silver + library + rubric │ │ │ │ Tools (DATA): │ │ read_scrape · list_images · classify │ │ read_file · write_file · diff │ │ sample_library(filter) │ │ render_preview (astro build) │ │ screenshot(page, viewport) ← NEW │ │ │ │ Tools (SKILLS): │ │ shape · critique · typeset · colorize │ │ layout · polish · bolder · quieter │ │ clarify · harden │ │ │ │ Loop: │ │ 1. plan → pick next action │ │ 2. call tool │ │ 3. observe (screenshot + vision) │ │ 4. critique (score against rubric) │ │ 5. stop if gate_pass or budget done │ └──────────────────────────────────────────────┘

Quality Rubric v1 Drafted

scripts/pipeline/rubric.json — 9 dimensions, weighted, with ship gate

The agent's grading function. Without this, "critique" is vibes and the loop can't converge. Designed around the impeccable skill's vocabulary so linked skills slot in cleanly.

#DimensionWeightLinked Skills
1distinctiveness / anti-slop1.5×boldercritiqueextract
2typography1.2×typesetpolish
3color & contrast1.0×colorizequieter
4spatial / layout1.2×layout
5information hierarchy1.3×shapeclarify
6UX writing1.0×clarifypolish
7imagery & asset fit1.1×polish
8trust signals (conversion)1.4×hardenpolish
9craft / polish1.0×polishharden

Ship gate: every dim ≥ 6, distinctiveness ≥ 7, trust_signals ≥ 7, overall ≥ 7.5.
Stop iterating when: gate passes, OR no improvement 2 iters in a row, OR 6-loop budget exhausted.

Why trust_signals is 1.4× and non-negotiable: for local-practice sites, a striking design that loses the visitor (phone buried, doctor invisible, no reviews) is a failure, not a win. The rubric makes that opinion explicit in math.

Non-negotiable design principles Keep these

  • Intelligent templates, not templated intelligence. Sections whose data is null must be excluded from layout, never rendered with dashes.
  • Divergence is a first-class requirement. The agent must be able to read its own past builds and be told to actively avoid them.
  • Skills, not prompts. Every design decision should be made by invoking a skill with a scope, not by writing a new prompt chain.
  • Data extraction stays linear. Scrape + silver + merge are solved problems — don't put the agent in charge of I/O. The agent owns design and composition only.
  • Trust outranks novelty for this vertical. Built into the rubric (trust_signals 1.4× weight, gate requires ≥ 7).
  • Every shipped build joins the library. Auto-distill on ship means the system gets smarter — and more divergent — with every practice we onboard.

How to trigger the agent Usage

# Full agentic run (build + agent loop, up to 6 iterations)
node scripts/pipeline/build-site.js \
  --url https://www.springstdentistry.com/ \
  --output /tmp/springst-build \
  --agent

# Cap iterations (faster for testing)
node scripts/pipeline/build-site.js \
  --url https://... --output /tmp/out \
  --agent --agent-iterations 3

# Or via env var (useful in CI)
GROUNDWORK_AGENT=1 node scripts/pipeline/build-site.js ...

Agent loop only fires when --agent is passed AND the Astro build succeeded AND ANTHROPIC_API_KEY is set. Safe to omit on linear runs.

Prerequisite: npm i -D playwright && npx playwright install chromium (observe.js requires Playwright).
